# Module 1 學員手卡 v1

## task brief 外層模板
### 現在發生了什麼
- 功能 / endpoint / 模組
- 環境
- 已知現象
- 已知訊號
- 背景變動

### 你現在要 AI 先做什麼
- 一次只交代一個工作階段

### 你希望它怎麼輸出
- 問題摘要
- 可能方向
- 還缺資訊
- 建議下一步

### 最後一句怎麼把 AI 收住
- Do not write code yet.
- Analyze before coding.
- If information is insufficient, tell me what is missing first.

## 自我檢查
1. 我寫的是現象，還是已經偷塞結論？
2. 我交代的是一個步驟，還是整包工作？
3. 我有沒有先指定輸出？
4. 我有沒有用最後一句把 AI 收在正確工作階段？


## 一頁式判斷提醒：先 Plan 還是先 Act
先不要急著叫 AI 動手。先問自己三題：

1. 這個問題現在還模糊嗎？
2. 這個任務如果直接改，風險會不會擴散？
3. 這題是不是可能需要更穩的能力、更多上下文、也更花成本？

如果上面有兩題以上答「是」，先用較保守的方式：
- 先分析
- 先整理缺口
- 先比較方向
- 暫時不要太快進 Act
