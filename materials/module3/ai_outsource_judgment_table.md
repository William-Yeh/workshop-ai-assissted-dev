# Module 3｜AI 可做 / 不可外包判斷表

## 類別 A｜AI 可以先做
適合先讓 AI 幫忙起草或整理：
- task brief 草稿
- context 缺漏清單
- symptom / candidate causes 拆分
- validation move 候選
- first-layer review note
- PR / issue summary 草稿

## 類別 B｜AI 可以先做草稿，但人一定要決定
- validation order 最終採納
- consumer expectation 判讀
- patch 是否真的合理
- style consistency 是否可接受
- test 是否已足夠
- short-term fix 與 long-term fix 的取捨

## 類別 C｜不應直接外包給 AI
- 最終風險承擔
- 規格 / 需求的正式拍板
- 高風險 migration 決策
- team-wide rule enforcement 設計
- 是否進入 workflow gate / automated rule / CI policy

## 講師提醒
這張表的目的，不是限制 AI。
這張表的目的，是讓團隊知道：
哪些事情可以先用 AI 降低整理成本，
哪些事情仍必須由人來負責判斷與承擔。
