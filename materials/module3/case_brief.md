# Module 3 Case Brief

## 模組定位
Module 3 的重點不是再教一次 prompt 技巧。
它要處理的是：
- 把 Module 1 與 Module 2 已證明有效的做法整理成團隊可共用格式
- 讓別人接手時，不必猜原作者腦中的隱含規則
- 讓 AI 協作從一次性成功，變成可複用、可交接、可討論的資產

## 本模組真正要解的問題
常見問題不是「大家都不會下 prompt」，而是：
- 個人有招，但別人接不起來
- 有人會做 validation order，但做法沒有固定欄位
- 有人會做 style consistency review，但沒有共用口徑
- 交接時，只留下 prompt，沒有留下使用條件、輸入需求、驗證點與停損線

## 本模組核心任務
把個人有效做法，整理成最少包含以下欄位的共用模板：
1. 用途 / 要解的問題
2. 何時用 / 何時不要用
3. 必要輸入
4. 最小足夠 context
5. 輸出格式
6. 驗證點 / handoff 點
7. AI 可做 / 不可外包的邊界

## 建議承接來源
優先從以下資產挑一個來模板化：
- Module 1 的 task brief 外層模板
- Module 2 的 validation order 做法
- Module 2 的 style consistency 自查口徑
- Module 2 的 first-layer quality delivery 寫法

## 本模組可交付物
以下擇一或組合完成：
- 團隊共用 task brief 模板
- 團隊共用 validation / diagnosis 模板
- 團隊共用 style consistency review 模板
- AI 可做 / 不可外包判斷表
- 模板交接 checklist

## 停損線
本模組停在：
- 模板化
- checklist 化
- 結構化輸出
- 可交接格式

本模組不要求：
- workflow gate
- team-level enforcement rule
- automated review rule
- CI / bot / policy integration

以上內容留到 Module 4。
