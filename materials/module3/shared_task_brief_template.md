# Team Shared Template｜Task Brief

## 這份模板用來做什麼？
把模糊工作整理成 AI 可協作、可檢查、可交接的任務。

## 何時用
- 任務已大致知道要解什麼
- 需要 AI 先整理、分析、草擬，而不是完全自由發揮
- 希望不同人交給 AI 時，至少有同一層基本格式

## 何時不要用
- 問題還太模糊
- 關鍵需求、邊界、consumer 還不清楚
- 任務本質需要先做 diagnosis，而不是先做 delivery

## 必要輸入
- 任務背景
- 本次要處理的範圍
- 相關檔案 / repo 區域
- 既有慣例或限制
- 預期輸出格式

## 最小足夠 context
- 只放本次決策需要的 domain 與檔案
- 明講哪些 consumer / flow 是本次判準
- 補上相鄰 code 的命名、結構、錯誤處理慣例

## 建議輸出格式
1. 先重述任務
2. 列出缺少但重要的資訊
3. 提出初步做法或草稿
4. 標出風險 / 假設 / 待驗點
5. 用指定格式交付

## final line 範例
- 先不要直接改 code，先指出仍缺哪些關鍵資訊。
- 先交 task brief 草稿，不要直接進完整 patch。
- 若你覺得需求仍模糊，先用條列列出待確認點。

## 最低驗證
- 任務有沒有被講成可協作的範圍？
- 有沒有把必要 context 與不必要 context 分開？
- 輸出是否可檢查，而不是只求看起來完整？

## handoff 提醒
留給下一位接手者：
- 這份 brief 依賴哪些假設
- 哪些地方尚未驗證
- 哪些地方不能把 AI 草稿直接當成定案
